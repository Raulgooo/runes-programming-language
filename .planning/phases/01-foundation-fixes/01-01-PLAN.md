<objective>
Add the TY_INFER_ERROR poison type to the Runes compiler type system and migrate all TY_UNKNOWN guards in typecheck.c to use a type_is_resolved() helper that suppresses checks for both TY_INFER_ERROR and TY_UNKNOWN.

Purpose: This is the prerequisite for all other Phase 1 fixes. Without TY_INFER_ERROR, failed type inference returns TY_UNKNOWN which silently passes all checks, hiding real errors. After this plan, the compiler can distinguish "not yet inferred" (TY_UNKNOWN) from "inference failed" (TY_INFER_ERROR) and suppress cascading errors on the latter.

Output: Modified types.h, types.c, typecheck.c with TY_INFER_ERROR integrated.
</objective>
